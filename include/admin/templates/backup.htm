Backup mit BigDump importieren

Beachten: es kann keine Garantie dafür übernommen werden, dass dieses Script immer einwandfrei funktioniert und alle Daten aus der Datenbank korrekt gespeichert werden!

Vor dem Wiedereinspielen dieses Backups müssen alle bestehenden Tabellen gelöscht werden damit sie vom Backup angelegt werden. (Option beachten)

Beim Einspielen mit phpmyadmin ist es wichtig, welche Kodierung gewählt wird, damit Umlaute/Sonderzeichen erhalten bleiben. Standard bei phpmyadmin ist UTF-8. Für BigDump ANSI benutzen.

Nur ein "Name" pro Tag, vorherige Backups vom gleichen Tag gehen also verloren.
(benötigt um mit ilch wieder einzuspielen)
Ausgabecodierung: ANSI / UTF-8
